# BUILD: Orders Subsystem

Mode: full (self-contained).

A coding agent builds this subsystem from this document plus the design/ tree.
The pack under design/pack/ is the parent's frozen interface; if it changes,
the parent reissues it and this design re-verifies (machinery check --gate g5).

## 1. What this is

The orders half of checkout-split: one service owning the Order entity,
coupled to the payments subsystem only through the three boundary events in
design/pack/events.md (request out; markPaid, markDeclined in).

## 2. Domain model

Source of truth: design/orders.modelith.yaml. The Order entity and OrderStatus
enum are the pack's frozen public shape; internal additions belong here, never
in the enum.

## 3. Behavior

design/machines/Order.machine.json is the lifecycle; the generated
design/machines/Order.oracle.md is the canonical transition test spec, keyed
by stable id. The contract refinement (design/formal/OrderPackRefinement.tla,
generated by machinery pack refine from packmap.yaml) proves this machine
refines the pack's OrdersContract; verify-formal TLC-checks it.

## 4. Traceability

| invariant | enforcement |
|---|---|
| no-ship-without-capture | structural: `Shipped` is reachable only from `Paid`, and `Paid` only on `markPaid`; see `_ignores` on Placed |
| order-total-positive | creation-time validation at the API boundary (attested; creation is not a machine transition) |

## 5. Test specification

Transition tests: every row of design/machines/Order.oracle.md, keyed on the
stable id column. Named-unit tests: the matrix table (a), with the outbox
`request` action as an integration test against the real outbox table.
Redelivery property test: any resting state receiving a duplicate settlement
event is unchanged (the `_ignores` reasons).

## 6. State migration

`Order.status` is persisted (placement table). No persisted instances yet; the
protocol applies from first deployment: any rename/split/removal of an
OrderStatus value ships with a mapping table from old persisted values to new
states, or an explicit drain rule, in this section. Note: the enum is the
pack's frozen shape, so such a change is a PARENT change that reissues packs.

## 7. Toolchain and versions

Go 1.26, stdlib testing; machinery oracle design/machines; machinery check
design (add --impl <dir> once code exists); machinery pack refine design;
machinery verify-formal design (Java for TLC).

## 8. Hard-TDD protocol

Test-writer derives tests from sections 4-5 keyed on oracle stable ids; tests
lock; the implementer makes them pass without editing them. A wrong test is a
design defect: fix the design, regenerate, rerun.
